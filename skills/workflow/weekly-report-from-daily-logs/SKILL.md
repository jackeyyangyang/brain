# Weekly Report From Daily Logs

从 `daily_logs/` 中读取一周范围内的日报，提炼“本周完成”和“下周计划”，并在 `daily_logs/` 下新建一个周报 `.txt` 文件。

## 适用任务

优先处理这类请求：
- 根据本周日报生成周报
- 从上周四到本周四汇总工作进展
- 输出给 Mentor/Leader 的中文周报模板
- 把 scattered daily logs 整理成一份可直接发送的周报

如果用户已经给了模板，就严格沿用模板口吻和结构。

## 周范围规则

默认按**周四到下周四**作为一周边界。

执行时按下面规则理解：
- 如果用户明确给出目标日期，围绕该日期所在的“周四结算周”取范围。
- 如果用户没给目标日期，优先用今天日期。
- 一份周报汇总区间为：**上一个周四（含）到当前周三/周四生成时可见的最近日志（含）**。
- 如果用户明确说“本周（从上周四到下周四为一周）”，按这个口径汇总，不自行改口径。

如果日志缺失，不编造内容；只基于现有日志总结，并在必要时简短说明“部分日期无日报记录”。

## 输入来源

1. 先查看 `daily_logs/` 下是否存在与本周范围匹配的 `YYYY-MM-DD.md`。
2. 优先读取范围内文件。
3. 如果某篇日报包含“同日补充材料”“commit 记录”“总结”等内容，可以吸收其中与本周工作直接相关的信息。
4. 忽略与周报目标无关的长篇附录、原始数据、重复段落。

## 提炼规则

### 本周完成

提炼 2 到 5 条，要求：
- 用结果导向表达，不要机械罗列所有改动
- 将同一主题的零碎工作合并
- 尽量保留项目内关键名词，如 `pairwise`、`benchmark`、`skill`、`render checkpoint`
- 优先写“完成了什么 / 改进了什么 / 解决了什么问题”

### 下周计划

提炼 2 到 4 条，来源优先级：
1. 日报中显式写出的“后续建议 / 后续事项 / 下周计划”
2. 从本周未完事项自然延伸出的下一步

如果日志里完全没有下周计划线索，再根据本周工作主线做最小推断，但不要发散。

## 输出风格

- 始终使用中文
- 语气自然、简洁、适合直接发给导师/Leader
- 不写“根据日志分析得出”之类元话术
- 不要输出 Markdown；最终文件内容是纯文本风格
- 保留阿拉伯数字编号
- 避免一条中过多细节，控制在一两句话内

## 文件命名

默认写到：
- `daily_logs/weekly_report_YYYY-MM-DD.txt`

其中日期默认使用周报生成日，或用户指定的目标日期。

如果用户明确要求别的名字，服从用户要求。

## 输出模板

严格按这个结构输出：

Mentor/Leader 们晚上好，
本周聚焦 SceneGen Agent 模块开发，进展如下：
【本周完成】
1. ...
2. ...
3. ...

【下周计划】
1. ...
2. ...
3. ...

感谢各位老师指导，祝工作顺利！
汇报人：jhyangyang
日期：YYYY 年 M 月 D 日

## 执行步骤

1. 确定目标周范围。
2. 找出 `daily_logs/` 下属于该范围的日报文件。
3. 读取并提炼“本周完成”。
4. 读取并提炼“下周计划”。
5. 按模板生成周报正文。
6. 在 `daily_logs/` 下创建 `.txt` 文件。
7. 向用户返回：
   - 使用了哪些日报
   - 生成的文件路径
   - 如有缺失日期，简短提示

## 质量检查

生成前自行检查：
- 是否混入了周范围之外的内容
- 是否把同一件事重复写了两次
- 是否存在过细实现细节，导致不像周报
- “本周完成”是否偏结果，“下周计划”是否偏行动
- 日期和文件名是否一致

## 示例

**输入意图：**
“根据这周的 daily logs，帮我生成一个给 Mentor/Leader 的周报 txt，放到 daily_logs 里。”

**预期行为：**
- 读取本周范围内的日报
- 归纳完成项与下周计划
- 新建 `daily_logs/weekly_report_YYYY-MM-DD.txt`
- 返回生成路径与简短说明
